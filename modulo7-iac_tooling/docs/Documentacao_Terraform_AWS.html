Documentação Terraform - AWS S3 + CloudFront

Este projeto Terraform cria uma infraestrutura segura na AWS para hospedar um site estático utilizando Amazon S3 e CloudFront com Origin Access Identity (OAI).

Estrutura do Projeto

O código está dividido em blocos principais:

  • Variáveis - Configurações parametrizáveis
  • Recursos (Resources) - Componentes da infraestrutura AWS
  • Outputs - Informações importantes após a criação

1. Provedor e Variáveis

Provedor AWS

Define o provedor da AWS e a região onde os recursos serão criados:

provider "aws" {
  region = var.aws_region
}

Variáveis Principais

  • aws_region: Região onde os recursos serão criados (padrão: us-east-1)
  • project_name: Nome base do projeto (padrão: jewelryan-app)

2. Criação do Bucket S3

Cria um bucket com nome baseado no projeto e um sufixo aleatório para garantir unicidade global:

resource "aws_s3_bucket" "website" {
  bucket = "${var.project_name}-${random_string.bucket_suffix.result}"
}
Nota: O sufixo aleatório garante que o nome do bucket seja único globalmente, já que nomes de bucket S3 devem ser únicos em toda a AWS.

3. Segurança com Origin Access Identity (OAI)

Bucket Privado

O bucket é configurado como completamente privado, bloqueando qualquer acesso público direto:

resource "aws_s3_bucket_public_access_block" "website" {
  bucket = aws_s3_bucket.website.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

Origin Access Identity

Cria uma identidade especial do CloudFront que terá permissão exclusiva para acessar o bucket:

resource "aws_cloudfront_origin_access_identity" "website" {
  comment = "OAI for ${var.project_name}"
}

Política do Bucket

Permite acesso apenas via CloudFront OAI (não mais acesso público):

resource "aws_s3_bucket_policy" "website" {
  bucket = aws_s3_bucket.website.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Sid    = "AllowCloudFrontOAI"
      Effect = "Allow"
      Principal = {
        AWS = aws_cloudfront_origin_access_identity.website.iam_arn
      }
      Action   = "s3:GetObject"
      Resource = "${aws_s3_bucket.website.arn}/*"
    }]
  })
}
Importante: Com OAI ativado, URLs diretas do S3 não funcionam mais. Todo acesso deve ser via CloudFront.

4. CloudFront Distribution

Cria uma distribuição CloudFront que serve como CDN global. Principais configurações:

resource "aws_cloudfront_distribution" "website" {
  enabled             = true
  is_ipv6_enabled     = true
  default_root_object = "index.html"

  origin {
    domain_name = aws_s3_bucket.website.bucket_regional_domain_name
    origin_id   = "S3-${aws_s3_bucket.website.id}"

    s3_origin_config {
      origin_access_identity = aws_cloudfront_origin_access_identity.website.cloudfront_access_identity_path
    }
  }

  default_cache_behavior {
    allowed_methods  = ["GET", "HEAD"]
    cached_methods   = ["GET", "HEAD"]
    target_origin_id = "S3-${aws_s3_bucket.website.id}"

    viewer_protocol_policy = "redirect-to-https"
    min_ttl                = 0
    default_ttl            = 3600  # 1 hora
    max_ttl                = 86400 # 24 horas
  }

  viewer_certificate {
    cloudfront_default_certificate = true
  }

  # Suporte para SPA (Single Page Application)
  custom_error_response {
    error_code         = 404
    response_code      = 200
    response_page_path = "/index.html"
  }

  custom_error_response {
    error_code         = 403
    response_code      = 200
    response_page_path = "/index.html"
  }
}

Características Principais:

  • HTTPS obrigatório - Redireciona HTTP para HTTPS automaticamente
  • Cache de 1 hora - Reduz requisições ao S3 e melhora performance
  • IPv6 habilitado - Suporte para protocolos modernos
  • SPA Support - Erros 404/403 redirecionam para index.html (Vue Router)
  • OAI configurado - Acesso seguro ao S3

5. Outputs

Após a execução, o Terraform exibe informações úteis:

output "s3_bucket_name" {
  description = "Nome do bucket S3"
  value       = aws_s3_bucket.website.id
}

output "cloudfront_domain" {
  description = "Domínio do CloudFront"
  value       = aws_cloudfront_distribution.website.domain_name
}

output "cloudfront_url" {
  description = "URL completa do CloudFront"
  value       = "https://${aws_cloudfront_distribution.website.domain_name}"
}

6. Uso Rápido

Inicializar o Terraform

cd iac/terraform
terraform init

Ver o plano de execução

terraform plan

Criar a infraestrutura

terraform apply -auto-approve

Destruir a infraestrutura

terraform destroy -auto-approve
Dica: Use o Makefile na raiz do projeto para comandos mais simples: make deploy, make destroy, etc.

Resumo dos Recursos

Recurso Descrição
S3 Bucket Armazenamento privado dos arquivos estáticos
CloudFront CDN global com HTTPS e cache
OAI Identidade para acesso seguro S3 → CloudFront
Bucket Policy Permite acesso apenas via OAI
Public Access Block Bloqueia completamente acesso público direto
Custom Error Response Suporte para aplicações SPA (Vue Router)

Requisitos

  • Terraform instalado (v1.0 ou superior)
  • Credenciais da AWS configuradas (via aws configure)
  • Permissões IAM para criar S3, CloudFront e IAM (para OAI)
