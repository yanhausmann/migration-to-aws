Documentação Scripts - Deployment AWS

Scripts automatizados para deploy da aplicação Vue.js na infraestrutura AWS. Disponível em Bash (Linux/Mac/WSL) e PowerShell (Windows).

Scripts Disponíveis

Bash setup.sh

Localização: iac/scripts/setup.sh

Função: Verifica e instala dependências necessárias para o projeto

Uso:

cd iac/scripts
./setup.sh

Bash deploy.sh

Localização: iac/scripts/deploy.sh

Função: Deploy completo automatizado (build + infraestrutura + upload)

Uso:

cd iac/scripts
./deploy.sh

PowerShell deploy.ps1

Localização: iac/scripts/deploy.ps1

Função: Deploy completo automatizado para Windows

Uso:

cd iac/scripts
.\deploy.ps1

setup.sh - Configuração do Ambiente

O que o script faz:

  • Cria arquivos de configuração do shell (.bashrc, .zshrc, .profile) se não existirem
  • Verifica se Node.js está instalado
  • Oferece instalação automática via nvm (se necessário)
  • Verifica se npm está disponível
  • Verifica se Terraform está instalado
  • Verifica se AWS CLI está instalado e configurado
  • Verifica se make está disponível
  • Exibe versões de todas as ferramentas

Verificações e Instalações:

Ferramenta Ação Instalação Automática
Node.js Verifica instalação Sim (via nvm)
npm Verifica instalação Vem com Node.js
Terraform Verifica instalação Sim (Linux)
AWS CLI Verifica instalação + credenciais Sim (Linux)
make Verifica instalação Sim (apt-get)
Dica: Execute este script antes de tentar fazer o deploy. Ele garante que todas as dependências estão instaladas.

deploy.sh / deploy.ps1 - Deploy Automatizado

Fluxo de Execução:

  1. Verificação de Dependências
    • Verifica se Node.js está instalado
    • Verifica se npm está instalado
    • Verifica se Terraform está instalado
    • Verifica se AWS CLI está instalado
    • Exibe versões de cada ferramenta
  2. Build da Aplicação
    • Executa npm install com tratamento de erros
    • Executa npm run build com verificação de sucesso
    • Valida se a pasta dist/ foi criada
  3. Inicialização do Terraform
    • Executa terraform init no diretório iac/terraform
  4. Aplicação da Infraestrutura
    • Executa terraform apply -auto-approve
    • Cria S3, CloudFront e OAI na AWS
  5. Upload dos Arquivos
    • Obtém o nome do bucket via terraform output
    • Sincroniza arquivos da pasta dist/ para o S3
    • Usa flag --delete para remover arquivos antigos
  6. Invalidação do Cache
    • Obtém o ID da distribuição CloudFront
    • Cria invalidação para o caminho /*
  7. Exibição da URL
    • Mostra a URL do CloudFront para acessar a aplicação

Tratamento de Erros:

Os scripts param a execução imediatamente se:

  • Alguma ferramenta necessária não estiver instalada
  • O build do npm falhar
  • A pasta dist/ não for criada
  • Não conseguir obter o nome do bucket
  • O upload para S3 falhar
Importante: Os scripts usam terraform apply -auto-approve, o que significa que não há confirmação manual. Revise suas configurações antes de executar.

Diferenças entre Bash e PowerShell

Aspecto Bash (deploy.sh) PowerShell (deploy.ps1)
Plataforma Linux, Mac, WSL Windows
Verificação de comandos command -v Get-Command
Verificação de pasta [ ! -d "dist" ] -not (Test-Path "dist")
Tratamento de erros set -e + if statements $ErrorActionPreference = "Stop" + try/catch
Exibição de versão $(node --version) $(node --version)
Cores no terminal Escape codes ANSI Write-Host -ForegroundColor

Exemplos de Uso

Primeira vez (Linux/Mac/WSL):

# 1. Configurar ambiente
cd iac/scripts
chmod +x *.sh
./setup.sh

# 2. Fazer deploy
./deploy.sh

Primeira vez (Windows PowerShell):

# 1. Permitir execução de scripts (executar como Admin)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

# 2. Fazer deploy
cd iac\scripts
.\deploy.ps1

Deploy subsequente:

# Linux/Mac/WSL
./deploy.sh

# Windows
.\deploy.ps1
Alternativa: Use o Makefile na raiz do projeto para simplificar: make deploy

iac/

Requisitos

  • Node.js 16+ e npm
  • Terraform 1.0+
  • AWS CLI configurado com credenciais válidas
  • Bash (Linux/Mac/WSL) ou PowerShell 5.1+ (Windows)
  • Permissões AWS para criar S3, CloudFront e IAM (OAI)

Solução de Problemas

Erro: "command not found" ou "não reconhecido"

Execute o setup.sh primeiro para instalar as dependências.

Erro: "npm install failed"

Verifique:

  • Conexão com internet
  • Permissões de escrita na pasta
  • Arquivo package.json existe

Erro: "AccessDenied" no S3

Configure suas credenciais AWS:

aws configure

PowerShell: "script não pode ser carregado"

Ajuste a política de execução:

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
