Migração Azure → AWS - Jewelry App

Resumo do Projeto

Migração de uma aplicação Vue.js de hospedagem Azure para AWS, implementando infraestrutura como código (IaC) com automação completa de deploy.

Objetivo

Reduzir custos operacionais migrando de Azure para AWS, aproveitando serviços gerenciados otimizados para sites estáticos.

Tecnologias Utilizadas

Frontend

  • Vue.js 3 - Framework JavaScript progressivo
  • Vite 4 - Build tool moderna e rápida

AWS Services

  • S3 - Storage de objetos para hospedar arquivos estáticos
  • CloudFront - CDN global com HTTPS incluído
  • OAI - Origin Access Identity para segurança (bucket privado)

Infraestrutura como Código

  • Terraform - Provisionamento declarativo da infraestrutura
  • Makefile - Automação de comandos de build e deploy
  • Bash/PowerShell - Scripts de deploy multiplataforma

Por Que AWS é Mais Barata?

Sites Estáticos = Custo Mínimo

  1. Sem servidores - Não há VMs rodando 24/7
  2. Pague pelo uso - Apenas por requisições e transferência
  3. CloudFront com free tier - 1TB/mês grátis no primeiro ano
  4. S3 altamente otimizado - Storage extremamente barato

Comparativo com Servidores Tradicionais

Recurso Servidor VM S3 + CloudFront
Custo fixo mensal $10-50 $0
Escalabilidade Manual Automática
Manutenção Alta Zero
Performance global Limitada CDN mundial

Estimativa de Custos AWS

Premissas:

  • Site com 100 páginas
  • Tamanho médio: 2MB por visita (HTML, CSS, JS, imagens)
  • Usuários navegam em média 5 páginas por sessão

Arquitetura Implementada

Usuario
↓

Benefícios de Segurança

  • Bucket S3 totalmente privado
  • Acesso apenas via CloudFront
  • HTTPS obrigatório
  • URLs diretas do S3 bloqueadas

Automação de Deploy

Comandos Disponíveis

make deploy      # Deploy completo (build + infra + upload)
make update      # Atualizar código (build + upload + cache)
make destroy     # Destruir infraestrutura
make help        # Ver todos os comandos

Fluxo de Deploy Automatizado

  1. Build - Compila aplicação Vue.js com Vite
  2. Terraform Init - Inicializa providers AWS
  3. Terraform Apply - Cria S3 + CloudFront + OAI
  4. Upload - Sincroniza arquivos para S3
  5. Cache Invalidation - Limpa cache do CloudFront

Tempo total: ~15-20 minutos (CloudFront leva tempo para propagar)

Vantagens da Migração

Operacionais

  • Deploy automatizado via Makefile
  • Infraestrutura versionada (Terraform)
  • Zero manutenção de servidores
  • Rollback facilitado via Git + Terraform

Performance

  • CDN global com 450+ pontos de presença
  • Cache inteligente no edge
  • HTTPS incluído sem custo extra
  • Latência reduzida globalmente

Financeiras

  • Redução de até 80% nos custos
  • Sem compromisso de longo prazo
  • Pague apenas pelo que usar
  • Free tier generoso (1TB/mês no 1º ano)

Conclusão

A migração de Azure para AWS para sites estáticos representa uma otimização significativa em:

  • Custo: Redução drástica por eliminar VMs
  • Escalabilidade: Automática e transparente
  • Manutenção: Praticamente zero
  • Performance: CDN global de alto desempenho

Para um e-commerce de joalheria com tráfego moderado (1.000-10.000 visitas/dia), o custo mensal fica entre $3-35/mês, comparado a $50-200/mês em hospedagem tradicional.
